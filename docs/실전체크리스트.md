# 🧳 실전 투입 체크리스트

여행 시작 전 프로덕션에 적용하는 순서. **마이그레이션·환경변수는 완료된 상태 전제.**

## 오늘(전날) 중
- [ ] Supabase 프로젝트 한 번 접속해 깨우기 (무료 플랜 ~1주 미사용 시 정지)

## 배포 순서 (순서 중요)
1. [ ] **`npm run prep:trip`** — 미리보기로 대상 목록 먼저 확인
   - 콘솔의 **실참가자 목록에 봇/테스트 계정이 섞이지 않았는지** 눈으로 확인
2. [ ] **`npm run prep:trip -- --yes`** — 실제 실행
   - 봇 삭제 + 이력 초기화 + 실참가자 전원 **초기 토큰 2,000** 지급
3. [ ] Supabase SQL Editor에서 마이그레이션 실행:
   - [ ] `0027_gacha_price.sql` (가챠 비용 200/+50)
   - [ ] `0028_role_abilities.sql` (도둑·해커·팀장 역할 + 능력)
4. [ ] `npm run seed:quiz` 실행 여부 확인 (퀴즈 100문제 데이터)

## 리허설 (봇 or 실계정 1~2명)
- [ ] 로그인 → 잔액 **2,000** 확인
- [ ] 효과카드 첫 유료 뽑기 비용 **200** 확인
- [ ] 관리자 대기실 → 팀 배정식 실행 → 각자 역할 카드 확인
  - [ ] 팀당 5명 = 스파이·광대·도둑·해커·팀장 1:1 배정되는지
- [ ] **도둑**: 대상 골라 훔치기 → 피해자 화면에 "도둑맞음" 토스트 뜨는지 / 같은 대상 재시도 차단
- [ ] **해커**: 해킹(100토큰) → 전원 잔액 리스트 + 10분 카운트다운 / 창 안 무료 재조회
- [ ] **팀장**: 팀명 변경이 대시보드에 반영 / 팀원 잔고 조회

## 여행 시작 시

## 알아둘 점
- 저녁게임 8종 허브(`dinner_contents/`)는 앱 배포에 포함 안 됨 → 노트북+모니터로 직접 진행.
- 섯다는 서버 타임아웃이 없어, 진행 중 최소 1명이 방 화면을 유지해야 자동 복구됨.
- 게임 정산은 되돌리기가 없으니 우승자 확인 후 정산.
